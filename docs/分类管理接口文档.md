# 文章分类与视频分类管理接口文档

## 1. 文章分类管理接口

### 1.1 分页查询文章分类

#### 接口地址
```
POST /article-categories/page
```

#### 请求方式
POST

#### 请求参数 (PageParam<ArticleCategoriesQuery>)
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | integer | 否 | 当前页码，默认为1 |
| pageSize | integer | 否 | 每页条数，默认为10 |
| query | object | 否 | 查询条件对象 |

##### query对象结构
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| name | string | 否 | 分类名称（模糊查询） |
| isEnabled | integer | 否 | 是否启用：0 否 1 是 |

#### 返回结果 (ResultBean<PageBean<ArticleCategoriesVO>>)
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "pageNum": 1,
    "pageSize": 10,
    "total": 100,
    "pages": 10,
    "list": [
      {
        "id": 1001,
        "name": "健康饮食",
        "description": "关于营养搭配、健康饮食习惯等方面的文章",
        "sortOrder": 1,
        "isEnabled": 1
      }
    ]
  }
}
```

## 5. 状态码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 操作成功 |
| 400 | 参数错误 |
| 404 | 数据不存在 |
| 500 | 服务器内部错误 |

## 6. 注意事项

1. 所有接口均采用RESTful风格设计
2. 所有请求和响应数据均为JSON格式
3. 删除操作为逻辑删除，不会真正从数据库中移除数据
4. 分类的isEnabled字段控制是否启用，0表示禁用，1表示启用
5. sortOrder字段用于控制分类在列表中的显示顺序，数值越小越靠前
6. 健康文章和视频的status字段含义：0 草稿 1 已发布 2 已下架 3 审核中 4 未通过审核
7. category字段存储的是JSON格式的分类ID数组
